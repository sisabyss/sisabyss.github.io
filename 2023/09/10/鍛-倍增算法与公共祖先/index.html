😎 Begin with a Question

废话少说，直接看题：

1483. 树节点的第 K 个祖先

给你一棵树，树上有 n 个节点，按从 0 到 n-1 编号。树以父
节点数组的形式给出，其中 parent[i] 是节点 i 的父节点。树的根节点是编号为 0 的节点。

树节点的第 k 个祖先节点是从该节点到根节点路径上的第 k 个节点。

实现 TreeAncestor 类：

  • TreeAncestor（int n， int[] parent） 对树和父数组中的节点数初始化对象。
  • getKthAncestor(int node, int k) 返回节点 node 的第 k 个祖先节点。如果不存在这样的祖先节点，返回 -1

示例：

输入：
[“TreeAncestor”,“getKthAncestor”,“getKthAncestor”,“getKthAncestor”]
[[7,[-1,0,0,1,1,2,2]],[3,1],[5,2],[6,3]]

输出：
[null,1,0,-1]

解释：
TreeAncestor treeAncestor = new TreeAncestor(7, [-1, 0, 0, 1, 1, 2, 2]);

treeAncestor.getKthAncestor(3, 1); // 返回 1 ，它是 3 的父节点
treeAncestor.getKthAncestor(5, 2); // 返回 0 ，它是 5 的祖父节点
treeAncestor.getKthAncestor(6, 3); // 返回 -1 因为不存在满足要求的祖先节点

/*
* Your TreeAncestor object will be instantiated and called as such:
* TreeAncestor* obj = new TreeAncestor(n, parent);
* int param_1 = obj->getKthAncestor(node,k);
*/

Hint:

  • 1 <= k <= n <= 5 * 10e4
  • parent[0] == -1 表示编号为 0 的节点是根节点。
  • 对于所有的 0 < i < n ，0 <= parent[i] < n 总成立
    0 <= node < n
  • 至多查询 5 * 10e4 次

😕思考：

最暴力的方法自然是，由 node 出发，一步步地往上跳，即：

$$
node→parent[node]→parent[parent[node]]→⋯
$$

需要 k 次才能跳到所需节点，时间复杂度为O(n)。

如何优化这个算法呢：
一个初步的想法是：预处理每个节点的“爷爷节点”，即父节点的父节点，那么就可以两次两次的往上跳，从而减少一半的跳跃次数（循环次数）。

进而，我们可以处理每个爷爷节点的爷爷结点，就可以四步四步的往上跳。

😮解惑：

预处理出每个节点的第 $2^i$ 个祖先节点，即第 $2,4,8,\cdots$ 个祖先节点（注意 $x$ 的第 $1$ 个祖先节点就是 $\textit{parent}[x]$。由于任意 $k$ 可以分解为若干不同的 $2$ 的幂（例如 $13=8+4+1$ ），所以只需要预处理出这些 $2^i$ 祖先节点，就可以快速地到达任意第 $k$ 个祖先节点。

😍算法：

在构造函数 TreeAncestor() 中，预处理每个节点 x 的 $2^i$ 祖先节点，记作 $pa[x][i]$ (若第
$2^i$ 个祖先不存在， 则 $pa[x][i] = -1$ )。计算方式如下：

  • 先枚举 $i$ 再枚举 $x$ ，相当于先算出所有的爷爷节点，再算出爷爷的爷爷节点，以此类推。

  • $pa[x][0] = parent[x]$, 即所有节点的父亲节点。

  • $pa[x][1] = pa[pa[x][0]][0]$, 即所有节点的爷爷节点。

  • 以此类推，$pa[x][i + 1] = pa[pa[x][i]][i]$, 即 $x$ 的第 $2^{i+1}$ 个祖先即为第 $2^i$ 个祖先的第 $2^i$ 个祖先节点。

  • 这里 $i + 1$ 最多为 $[log_2n]$ 。例如 $n = 13$ 时，$[log_2n] = 3$，至多需要处理第 $2^3$ 个祖先即可。（当然，可以把树高，或者每个节点的深度求出来，再据此做精细计算。）

对于 getKthAncestor，需要找到 k 的二进制表示中的所有 1（相当于把 k 分解为若干 $2^i$ ）。可以从小到大枚举 i，如果 k 右移 i 位后的最低位为 1，就说明 k 的二进制从低到高第 i 位是 1，那么往上跳 $2^i$ 步，将 node 更新为 pa[node][i]。如果 node=−1 则说明第 k 个祖先节点不存在。

复杂度分析

时间复杂度：预处理 $O(nlogn)$，回答每个询问 $O(logk)$ 。
空间复杂度：预处理需要 $O(nlogn)$ 的空间。

注：利用长链剖分，可以做到预处理 $O(nlogn)$ ，回答每个询问 $O(1)$ 的时间复杂度。

Tips

  1. 可将$pa[x][i]$的结构转为$pa[i][x]$,其能优化缓存，可能略微提升性能。
  2. __builtin_clz(x)可以获取$x$的二进制长度，但是坑点在于其只能获取int的二进制长度。倘若需要
    long long的则需要调用__builtin_clzll，否则会报错。

😗模板：最近公共祖先

如何计算树上任意两点x和y的最近公共祖先 $lca$ 呢？

设节点i的深度为 $depth[i]$ 。这用一次DFS即可预处理出来。

如果x的深度大于y，即 $depth[x] > depth[y]$ ，我们可以将x和y交换一下，便于我们的讨论。

我们将靠下的y更新为第 $depth[y] - depth[x]$ 个祖先节点，这样 x 和 y 就处于同一深度了。

如果此时$x = y$，那么 x 就是我们想要的 $lca$ ，否则,把 $x$ 和 $y$ 一起往上跳。

  • 如果 $x$ 的第 $2^i$ 个祖先节点不存在，即 $pa[x][i]=−1$ ，说明步子迈大了，将 $i$ 减 $1$ ，继续循环。
  • 如果 $x$ 的第 $2^i$ 个祖先节点存在，且 $pa[x][i]≠pa[y][i]$ ，说明 $lca$ 在 $pa[x][i]$ 的上面，那么更新 $x$ 为 $pa[x][i]$ , 更新 $y$ 为 $pa[y][i]$ ，将 $i$ 减 $1$，继续循环。否则，若 $pa[x][i]=pa[y][i]$，那么 $lca$ 可能在 $pa[x][i]$ 下面，由于无法向下跳，只能将 $i$ 减 $1$，继续循环。

注：你也可以用二分来理解上述算法。在 $x$ 到根节点的这条路径上猜一个点 $z$ 当作 $lca$，且 $x$ 与 $z$ 相距 $2^i$ 步。那么把 $x$ 和 $y$ 同时向上跳 $2^i$ 步，如果 $x≠y$ ，就说明 $lca$ 在 $z$ 的上面，否则 $lca$ 要么是 $z$，要么在 $z$ 的下面。这样一种二段性既说明了二分的正确性，又说明了每次上跳之后，步长一定要减半（类比二分查找，把搜索的区间长度减半）。

考虑到通常题目是用 $edges$ 的方式输入的，所以下面的模板先用 $edges$ 建图，再用 DFS 预处理。

复杂度分析

  • 时间复杂度：预处理 $O(nlogn)$，回答每个询问 $O(logn)$。
  • 空间复杂度：预处理需要 $O(nlogn)$ 的空间。
